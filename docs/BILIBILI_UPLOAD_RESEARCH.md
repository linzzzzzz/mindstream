# Bilibili Auto-Upload Research Report

**Date:** 2026-02-23  
**Purpose:** Explore options for automatically uploading AI-generated video clips to Bilibili

---

## Executive Summary

There are **existing open-source tools** that can handle Bilibili uploads. The most promising options are:

| Tool | Stars | Language | Best For |
|------|-------|----------|----------|
| **biliup** | 6.4k+ | Rust + Python | Dedicated Bilibili uploader |
| **social-auto-upload** | 1.5k+ | Python | Multi-platform (Bilibili + others) |

Both tools are **actively maintained** and support programmatic upload.

---

## Option 1: biliup (Recommended)

### Overview
- **GitHub:** https://github.com/biliup/biliup
- **Stars:** 6.4k+
- **Language:** Rust (core) + Python (engine) + Next.js (UI)
- **License:** MIT

### Features
- ✅ Command-line upload to Bilibili
- ✅ Web UI (port 19159)
- ✅ Multiple login methods: SMS, password, QR code, cookie
- ✅ Multi-part video support
- ✅ Live streaming recording
- ✅ 20+ platform support (Douyu, Huya, Twitch, etc.)

### CLI Usage
```bash
# Install
uv tool install biliup

# Login (saves to cookies.json)
biliup login

# Upload video
biliup upload video.mp4 --title "My Video" --desc "Description" --tag "AI,Tech"

# Start web server for programmatic access
biliup server --auth
# Then access http://localhost:19159
```

### Pros
- Dedicated to Bilibili
- Well-maintained
- Web UI for easy management
- Supports edge cases (multi-part, cover images)

### Cons
- Documentation is primarily in Chinese
- Requires Bilibili account credentials

- Authentication via cookies

### Pros

- Custom workflow

### Cons

## Recommended Approach

### For MVP: Use biliup

1. **Install biliup** on your server
2. **Configure login** (QR code or cookies)
3. **Integrate** with MindStream:
   ```python
   import subprocess
   subprocess.run([
       'biliup', 'upload', 
       'clip.mp4',
       '--title', 'AI Highlights: Sam Altman',
       '--desc', 'Generated by MindStream',
       '--tag', 'AI,Tech'
   ])
   ```

### For Multi-Platform: Use social-auto-upload

If you plan to also upload to:
- TikTok
- YouTube Shorts
- Xiaohongshu

Then `social-auto-upload` is better.

## Authentication Requirements

Both tools require:
1. **Bilibili account** with creator permissions
2. **Login method:**
   - QR code scan (easiest)
   - Cookies (most programmatic)
   - SMS/email login

### Cookie Flow
```
1. User logs in via QR code / credentials
2. Tool saves cookies to file (cookies.json)
3. Subsequent uploads use saved cookies
4. Periodic refresh needed (tools handle this)
```

## Integration with MindStream

### Architecture
```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│  MindStream    │────▶│  biliup CLI    │────▶│  Bilibili      │
│  (generates    │     │  (uploads)     │     │  (hosts clips) │
│   clips)       │     │                │     │                │
└─────────────────┘     └─────────────────┘     └─────────────────┘
```

### Implementation Steps
1. Install biliup on the server
2. Run initial login to authenticate
3. Create a wrapper function:
   ```python
   def upload_to_bilibili(video_path, title, description, tags):
       result = subprocess.run([
           'biliup', 'upload', video_path,
           '--title', title,
           '--desc', description,
           '--tag', ','.join(tags)
       ], capture_output=True, text=True)
       return result.stdout
   ```
4. Call this function after OpenClip generates clips
5. Get the Bilibili URL from output
6. Push URL to Telegram users

## Risks & Considerations

| Risk | Mitigation |
|------|------------|
| Account ban | Don't over-upload (1-2 per day max) |
| Copyright | Clips are transformative (fair use) |
| API changes | Use well-maintained tools |
| Rate limits | Add delays between uploads |
